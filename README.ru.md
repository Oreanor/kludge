# Kludge Code

**AI-ассистент прямо в VS Code — чат, редактирование кода, превью, git и автоматизация в одной боковой панели.**

Kludge Code подключается к AI-провайдерам, которые ты уже используешь (или запускаешь локально), и встраивает полноценный чат прямо в VS Code. Никаких браузерных вкладок, никакого копипаста — просто сайдбар, где ты разговариваешь с кодом.

## Возможности

### Несколько AI-провайдеров — твои ключи, твой выбор

Подключи любую комбинацию из 8 провайдеров одновременно. Kludge автоматически выбирает лучшую доступную модель, или выбирай сам из выпадающего списка.

| Провайдер | Модели |
|---|---|
| Google Gemini | Gemini 2.0 Flash, Gemini 1.5 Pro/Flash |
| Anthropic | Claude 3.5 Sonnet, Claude 3 Haiku |
| OpenAI | GPT-4o, GPT-4o Mini |
| Groq | Llama 3, Mixtral (сверхбыстрый инференс) |
| OpenRouter | Любая модель через единый API |
| DeepSeek | DeepSeek Chat, DeepSeek Coder |
| Mistral | Mistral Large, Codestral |
| Ollama | Любая локальная модель на твоей машине |

API-ключи хранятся в зашифрованном `SecretStorage` VS Code — никогда в открытом тексте.

### Чат с контекстом рабочего пространства

Каждое сообщение автоматически дополняется контекстом:

- **Активный файл** — файл, который ты редактируешь, всегда в контексте
- **Список файлов проекта** — AI знает, что есть в проекте
- **Выбор области (scope)** — сузь контекст до конкретного файла, папки или всего проекта
- **Двухпроходное чтение файлов** — если AI нужно содержимое файла, он запрашивает его и получает ответ уже с реальным кодом, без лишнего расхода токенов на нерелевантные файлы

---

### Несколько сессий чата

Работай над несколькими задачами параллельно, не теряя историю. Каждая сессия — отдельная вкладка вверху панели. Создавай, переключайся и закрывай независимо. Одна сессия может стримить, пока ты читаешь другую. Активная работа помечается точкой-индикатором.

---

### Снапшоты и восстановление

Перед каждым отправленным сообщением Kludge незаметно сохраняет снапшот всех открытых файлов рабочего пространства. Если ответ AI пошёл не туда — нажми **↩** рядом с любым прошлым сообщением, и файлы восстановятся до состояния перед этим запросом. Без git, без stash, без переключения веток.

Хранится до 15 снапшотов в globalState VS Code.

---

### Быстрые промпты

Действия одним кликом для типовых задач. Встроенные: рефакторинг, объяснить, написать тесты и др. Создавай свои:

- Выбери **промпт** из выпадающего списка
- Выбери **область** (активный файл / папка / весь проект)
- Нажми **＋** для мгновенного запуска или переключись в режим **Задача** для планирования

Кастомные промпты сохраняются в настройках VS Code (`kludge.customPrompts`) и синхронизируются через Settings Sync.

---

### Планировщик задач

Нужно, чтобы AI запустил задачу в определённое время? Запланируй любой промпт через встроенный календарь:

- Дата и время выбираются через пикер
- В календаре задачи отображаются **синими точками** (ожидают) и серыми (выполнены)
- Кликни на день — увидишь задачи с временем и превью текста
- Задачи переживают перезапуск VS Code — восстанавливаются и перезапускаются при активации
- Отменить любую задачу — один клик

### Панель git

Компактный git-тулбар прямо в зоне ввода:

| Кнопка | Действие |
|---|---|
| Выпадающий список веток | Переключение или создание новой ветки прямо здесь |
| Commit | AI автоматически генерирует сообщение коммита |
| Push | Коммит + пуш одной кнопкой |
| ↩prev | `git reset --hard HEAD~1` |
| ↩remote | `git fetch && git reset --hard origin/<branch>` |
| Init | `git init`, если репозитория ещё нет |

AI тоже может запускать git-операции сам. Когда просишь «закоммить это» или «запуши изменения», он добавляет в ответ скрытые `<vscode-cmd>` теги, которые исполняются автоматически и не показываются в чате.

### npm-скрипты

Запускай любой скрипт из `package.json` прямо из панели. Список скриптов определяется автоматически. Вывод идёт в терминал VS Code.

---

### Превью с пикером элементов

Открой запущенный dev-сервер (Vite, Next.js, React, Angular, Vue) в боковой панели, не покидая VS Code:

- **Автоопределение** популярных портов (5173, 3000, 4200, 8080 ...) — если открыто несколько, выбираешь сам
- Прозрачный прокси внедряет bridge-скрипт в каждую загрузку страницы
- **Пикер элементов** — кликни на любой элемент в превью, и его селектор, тег, размеры и вычисленные стили попадают чипом в поле ввода чата — удобно для точечного редактирования стилей
- **Авто-фикс runtime ошибок** — если в превью срабатывает `console.error`, Kludge автоматически отправляет ошибку и стектрейс в AI и стримит предложение по фиксу
- **Авто-обновление** при сохранении файла

### Интеграция с Telegram

Управляй ассистентом с телефона. Подключи токен Telegram-бота и chat ID — в боковой панели появится вкладка **Telegram**:

- Сообщения из Telegram отображаются в реальном времени
- AI отвечает, и ответ автоматически отправляется обратно в Telegram
- Токен хранится в зашифрованном хранилище VS Code; отображается как `••••••••••••`
- Кнопка подключения неактивна, пока ты реально ничего не изменил

---

### Рендеринг Markdown

Ответы AI отображаются как форматированный markdown — блоки кода, списки, жирный текст, цитаты, инлайн-код. Блоки кода **сворачиваются**: длинные сниппеты свёрнуты по умолчанию, разворачиваются по клику — читаемость сохраняется даже с большими диффами.

---

### Управление провайдерами

Добавляй, удаляй, включай и выключай любой провайдер прямо в VS Code:

- Ключи показываются замаскированными (`••••••••••••`)
- При удалении ключ переходит в **отложенное состояние** с кнопкой Restore — случайное удаление легко откатить
- Отключение провайдера убирает его модели из авто-выбора, не удаляя ключ

## Быстрый старт

1. Установи расширение
2. Открой панель **Kludge Code** на Activity Bar (иконка огня)
3. Разверни **Providers** и добавь хотя бы один API-ключ
4. Начинай общаться

Для Ollama — установи локально, запусти и добавь `http://localhost:11434` как URL Ollama. API-ключ не нужен.

|---|---|

## Настройки

| Настройка | Тип | Описание |
|---|---|---|
| `kludge.customPrompts` | array | Кастомные быстрые промпты. Каждый элемент: `label` (название в UI) и `text` (текст промпта). |

```json
"kludge.customPrompts": [
  {
    "label": "Написать тесты",
    "text": "Напиши исчерпывающие юнит-тесты для выбранного файла. Используй тот же тестовый фреймворк, который уже есть в проекте."
  },
  {
    "label": "Ревью безопасности",
    "text": "Проверь этот код на уязвимости. Ищи инъекции, проблемы с авторизацией, небезопасную десериализацию и другие риски из OWASP top 10."
  }
]
```
